## 5.2 Computational Graphs

### Definition and Structure

A **computational graph** is a directed acyclic graph (DAG) that represents a mathematical expression:

  * **Leaf nodes** : inputs and parameters (x\mathbf{x}x, W\mathbf{W}W, b\mathbf{b}b, yyy) ‚ÄĒ no incoming edges
  * **Interior nodes** : intermediate computed values (matrix products, additions, activations)
  * **Directed edges** : data flow from inputs to outputs
  * **Root node** : the final scalar loss LLL



Every modern deep learning framework (PyTorch, JAX, TensorFlow) builds this graph implicitly during the forward pass and uses it to compute gradients during the backward pass.

### Local Gradient Table

At each node, backpropagation needs two things: the **upstream gradient** (what ‚ąāL/‚ąāoutput\partial L / \partial \text{output}‚ąāL/‚ąāoutput is), and the **local gradient** (‚ąāoutput/‚ąāinput\partial \text{output} / \partial \text{input}‚ąāoutput/‚ąāinput). The downstream gradient is their product.

The local gradient depends only on the operation at that node ‚ÄĒ it can be computed once and stored:

Operation | Forward | Local gradient  
---|---|---

Multiplication: z=xyz = xyz=xy | z=xyz = xyz=xy | ‚ąāz/‚ąāx=y\partial z/\partial x = y‚ąāz/‚ąāx=y, ‚ąāz/‚ąāy=x\partial z/\partial y = x‚ąāz/‚ąāy=x

Three patterns to note:

**Addition is a gradient distributor** : it passes the upstream gradient unchanged to both inputs, since ‚ąāz/‚ąāx=‚ąāz/‚ąāy=1\partial z/\partial x = \partial z/\partial y = 1‚ąāz/‚ąāx=‚ąāz/‚ąāy=1.

**Multiplication is a gradient swapper** : the gradient for xxx involves yyy and vice versa ‚ÄĒ ‚ąāz/‚ąāx=y\partial z/\partial x = y‚ąāz/‚ąāx=y, ‚ąāz/‚ąāy=x\partial z/\partial y = x‚ąāz/‚ąāy=x. The gradient for each input is the upstream gradient multiplied by the _other_ input's value.

**Multi-output nodes sum gradients** : if a node's output feeds into kkk downstream nodes, the upstream gradient it receives is the sum of contributions from all kkk paths (multivariate chain rule).

### Forward-Backward Symmetry

The computational graph reveals an elegant symmetry between the forward and backward passes:

| Forward pass | Backward pass  
---|---|---  
Direction | Leaf ‚Üí Root | Root ‚Üí Leaf  
Computes | Node values | Node gradients ‚ąāL/‚ąā‚čÖ\partial L/\partial \cdot‚ąāL/‚ąā‚čÖ  
Local operation | Apply node's function | Multiply by local gradient  
Dependencies | Needs parent values | Needs child gradient + cached parent values  
  
The backward pass runs in the _reverse_ topological order of the forward pass. This is why deep learning frameworks cache intermediate activations during forward computation ‚ÄĒ they are needed again to compute local gradients in the backward pass.
