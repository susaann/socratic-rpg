## Exercises

### Concept Check

**4.1** Give an example of a two-dimensional classification problem that a single-layer perceptron cannot solve, and explain geometrically why.

Answer

**XOR** is the canonical example: inputs (0,0)(0,0)(0,0) and (1,1)(1,1)(1,1) belong to class 0; inputs (0,1)(0,1)(0,1) and (1,0)(1,0)(1,0) belong to class 1.

**Geometric explanation** : a single-layer perceptron classifies by finding a line (in R2\mathbb{R}^2R2) that separates the two classes. The class-0 points (0,0)(0,0)(0,0) and (1,1)(1,1)(1,1) lie on opposite corners of the unit square; the class-1 points (0,1)(0,1)(0,1) and (1,0)(1,0)(1,0) lie on the other two corners. No single straight line can separate the diagonal pairs — the two classes are interleaved and not linearly separable.

Any ddd-dimensional single-layer classifier is restricted to hyperplane decision boundaries. Problems with curved or multi-region boundaries (including XOR, rings, spirals, concentric circles) cannot be solved by any single linear layer.

**4.2** Why does the vanishing gradient problem worsen with depth for sigmoid activations? Estimate the gradient attenuation through a 10-layer sigmoid network.

Answer

The maximum value of σ′(z)=σ(z)(1−σ(z))\sigma'(z) = \sigma(z)(1-\sigma(z))σ′(z)=σ(z)(1−σ(z)) is 0.250.250.25, achieved at z=0z = 0z=0. At saturation (∣z∣|z|∣z∣ large), σ′(z)→0\sigma'(z) \to 0σ′(z)→0.

During backpropagation, the gradient passes through each layer's activation derivative via the chain rule. In the best case (every neuron exactly at z=0z = 0z=0), the gradient through 9 sigmoid layers (for a 10-layer network, output layer to first hidden layer) attenuates by:

0.259≈3.8×10−60.25^9 \approx 3.8 \times 10^{-6}0.259≈3.8×10−6

In practice, many neurons are in saturation regions where σ′≪0.25\sigma' \ll 0.25σ′≪0.25, making attenuation far worse. This means parameters in early layers receive gradient signals millions to billions of times weaker than those in the output layer, making them effectively frozen and unlearnable. This was the primary obstacle to training deep networks before ReLU.

**4.3** ReLU's derivative is 1 for z>0z > 0z>0 and 0 for z<0z < 0z<0. Does this mean gradient signals through a 10-layer ReLU network do _not_ attenuate? What actually happens?

Answer

**Partially correct — but gradient flow in ReLU networks is more nuanced.**

For neurons with z>0z > 0z>0 (active neurons), the ReLU derivative is exactly 1, so gradients pass through without attenuation. This is what makes ReLU superior to sigmoid for deep networks.

**However** , ReLU networks still face gradient issues:

  1. **Dead ReLU neurons** : neurons with z≤0z \leq 0z≤0 have derivative 0, completely blocking gradient flow through those paths. If many neurons are dead, gradient information cannot reach early layers through those neurons.

  2. **Gradient depends on the fraction of active neurons** : in a typical ReLU network, roughly half the neurons are active (positive pre-activation) and half are inactive. Effective gradient transmission requires that at least one path of active neurons connects output to input.

  3. **Exploding gradients** : without saturation, gradients can grow unboundedly through many active layers. Deep ReLU networks require gradient clipping or careful initialization (He initialization, Chapter 6) to prevent gradient explosion.




In practice, ReLU solves vanishing gradients in moderate depth (10–100 layers) but residual connections (Chapter 9) are needed for very deep networks (100–1000 layers).

**4.4** State the three things the Universal Approximation Theorem guarantees and three things it does _not_ guarantee.

Answer

**Does guarantee** :

  1. For any continuous function fff on a compact set and any precision ε>0\varepsilon > 0ε>0, a single-hidden-layer neural network with the appropriate activation function (any non-polynomial measurable function) exists that approximates fff within ε\varepsilonε
  2. The approximating network has a finite number of neurons NNN
  3. The result holds for any input dimension nnn



**Does not guarantee** :

  1. **Size bounds** : the required NNN may be exponentially large (curse of dimensionality for naive constructions)
  2. **Findability** : gradient descent on the non-convex loss surface may not find the approximating parameters, even though they exist
  3. **Generalization** : a network that approximates fff on training data does not necessarily generalize to new inputs (overfitting)

**4.8** Consider the statement: "More hidden neurons always improves the network's approximation ability." Is this true? Discuss what happens to the bias-variance tradeoff as the number of hidden units increases, and what practical intervention can prevent the associated failure mode.

Answer

**Partially true but incomplete** : adding hidden neurons expands the hypothesis space H\mathcal{H}H, which can reduce bias (the model can represent more complex functions). However, a larger hypothesis space also increases variance — the model is more sensitive to the specific training set, and with enough neurons, any training set can be memorized exactly (interpolated), leading to poor generalization.

**Bias-variance analysis** :

  * **Bias** : strictly non-increasing in the number of neurons, since a larger H\mathcal{H}H contains a strictly wider set of functions. Adding neurons can never make the best achievable approximation worse.
  * **Variance** : generally increasing in the number of neurons. More parameters means more degrees of freedom, so the learned function varies more across different training sets.



**What goes wrong** : a network with, say, 10,000 neurons trained on 100 examples can perfectly memorize the training labels. The training loss reaches zero, but the test loss is poor — the network has fit training noise, not the underlying function.

**Practical interventions** :

  1. **L2 / weight decay regularization** : equivalent to a Gaussian prior on weights (Chapter 2), constraining the effective complexity even as neuron count increases
  2. **Dropout** (Chapter 6): randomly zeroing activations during training, which prevents co-adaptation and effectively trains an ensemble of smaller networks
  3. **Early stopping** : monitor validation loss and stop training when it starts to increase, before the network fully memorizes training noise
  4. **Data augmentation** : artificially expanding the training set reduces the effective variance of the learned function



The correct design principle is: choose network size large enough to have low bias, then control variance with regularization — rather than limiting network size to control variance.
