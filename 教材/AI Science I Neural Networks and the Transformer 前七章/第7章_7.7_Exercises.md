**1.** A 20-layer sigmoid network is trained, and you observe that the weights in layers 1–5 barely change after hundreds of epochs, while layers 15–20 learn effectively. What is the most likely explanation? Which term in the gradient product ∏kdiag(σ′(z(k)))W(k)\prod_k \text{diag}(\sigma'(\mathbf{z}^{(k)})) \mathbf{W}^{(k)}∏k​diag(σ′(z(k)))W(k) is responsible?

The responsible term is diag(σ′(z(k)))\text{diag}(\sigma'(\mathbf{z}^{(k)}))diag(σ′(z(k))) — the activation derivative diagonal matrix in each layer. This is independent of the weight matrices.

**2.** Explain why a BN layer does not need a bias term b\mathbf{b}b in the preceding linear layer.

Answer

Batch Normalization subtracts the mini-batch mean μB\mu_\mathcal{B}μB​ from all pre-activation values Wh+b\mathbf{Wh} + \mathbf{b}Wh+b. When computing z−μBz - \mu_\mathcal{B}z−μB​, the constant bias b\mathbf{b}b cancels: (Wh+b)−μB=Wh+b−(Wh‾+b)=Wh−Wh‾(Wh + b) - \mu_\mathcal{B} = Wh + b - (\overline{Wh} + b) = Wh - \overline{Wh}(Wh+b)−μB​=Wh+b−(Wh+b)=Wh−Wh. The bias contribution to the mean is subtracted away. The role of the bias (shifting the activation distribution) is then performed by the learned β\betaβ parameter in the BN scale-and-shift step, which is strictly more expressive since β\betaβ is learned by gradient descent.

**3.** A model achieves 99% accuracy on training data and 52% on validation data (chance level is 50% for binary classification). What is the diagnosis, and name two interventions ranked by what to try first.

Answer

Severe overfitting (high variance). The model has memorized training labels but learned no generalizable signal. Ranked interventions: (1) Add dropout or increase weight decay to constrain model complexity — address the regularization gap directly. (2) Collect more training data, which reduces variance without increasing bias. Other valid interventions: early stopping, smaller model architecture, data augmentation.

**6.** Sketch the training and validation loss curves for each of the following scenarios, and state the diagnosis:

(a) After 50 epochs, training loss = 0.05, validation loss = 0.50. (b) After 50 epochs, training loss = 1.80, validation loss = 1.85 (cross-entropy loss; chance is ~2.3 for 10 classes). (c) After 50 epochs, training loss is still decreasing from 0.8 to 0.2; validation loss decreased to 0.3 at epoch 30 but has since increased to 0.45.

Answer

(a) Large train-validation gap with low training loss: **severe overfitting**. The model has memorized training data. Prescription: stronger regularization (increase dropout rate or weight decay), data augmentation, or reduce model size.

(b) Both losses are high and similar: **underfitting** (high bias). The model lacks capacity to learn the data distribution. Prescription: increase model depth/width, train longer, reduce regularization, check for bugs in the training pipeline.

(c) Classic overfitting trajectory. Validation loss minimum was at epoch 30 (0.30). Early stopping should have triggered around epoch 30 (with patience of ~5–10 epochs). The returned model should be the checkpoint from epoch 30, not the final epoch. This illustrates why we must save the best checkpoint, not just the final one.

### Think Deeper

**7.** Batch Normalization makes the normalization statistics (μB\mu_\mathcal{B}μB​ and σB2\sigma_\mathcal{B}^2σB2​) depend on the entire mini-batch. This means the loss computed on example iii depends on the other examples in the same mini-batch.

(a) Why does this create an issue for the usual stochastic gradient descent convergence guarantees? (b) Is this dependence harmful or helpful in practice, and why?

Answer

(a) Standard SGD convergence proofs (e.g., those in Chapter 3) assume that the gradient of the loss on example iii, ∇θℓ(xi,yi;θ)\nabla_\theta \ell(x_i, y_i; \theta)∇θ​ℓ(xi​,yi​;θ), is an unbiased estimator of the true gradient ∇θL(θ)\nabla_\theta L(\theta)∇θ​L(θ). With BN, the loss on example iii includes contributions from the statistics of the other examples in the batch — the gradient ∇θℓ(xi,yi∣{xj}j≠i;θ)\nabla_\theta \ell(x_i, y_i | \\{x_j\\}_{j \neq i}; \theta)∇θ​ℓ(xi​,yi​∣{xj​}j=i​;θ) is no longer a pure function of (xi,yi)(x_i, y_i)(xi​,yi​). The unbiasedness argument breaks. The mini-batch gradient is still an unbiased estimator of the full-batch gradient (because the expectation over batch composition averages out the cross-example effects), but the within-batch correlation inflates variance and complicates the convergence analysis.

(b) Helpful in practice, for two reasons. First, the noise introduced by varying batch statistics acts as a regularizer — similar to the noise injection in Dropout — and reduces overfitting. Second, the normalization keeps activation magnitudes well-behaved throughout training, enabling much larger learning rates (10× or more) than are safe without BN. The practical speed-up from using larger learning rates far outweighs any theoretical inconvenience from breaking the independence assumption.

**8.** Early stopping is often described as "the poor man's regularization" because it requires no new hyperparameters beyond training time. However, it implicitly requires a validation set held out from training.

(a) If you use the same data for both model selection (via early stopping) and hyperparameter tuning (e.g., selecting the optimal learning rate), does the final test set estimate remain unbiased? Explain. (b) Propose a data splitting strategy for a dataset of 10,000 examples that uses early stopping, tunes 3 hyperparameters, and requires an unbiased final test estimate.

Answer

(a) No. If the same validation set is used for both early stopping (choosing t∗t^*t∗) and hyperparameter selection (choosing learning rate, dropout rate, etc.), then the validation set has been used multiple times to make training decisions. The validation loss at the selected configuration is a biased (optimistic) estimate of generalization performance — you have, in effect, fit the validation set. The test set estimate would remain unbiased only if the test set has never been used in any decision.

(b) A proper three-way split: (1) **Training set** (7,000 examples): used for gradient descent. (2) **Validation set** (1,500 examples): used for early stopping during each training run, and for comparing different hyperparameter configurations. (3) **Test set** (1,500 examples): held out completely until the final selected model is chosen. The procedure: run training with each hyperparameter configuration using early stopping on the validation set → select the configuration with the best validation performance → train the final model on train+validation with the selected hyperparameters → evaluate once on the test set. The test set is touched exactly once.
