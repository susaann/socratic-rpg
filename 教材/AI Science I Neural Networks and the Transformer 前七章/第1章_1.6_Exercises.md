## 1.6 Exercises

### Concept Check

**Q1.** Explain the AI ⊃ ML ⊃ DL ⊃ LLM hierarchy. Give an example of an AI system that is _not_ an ML system, an ML system that is _not_ a DL system, and a DL system that is _not_ an LLM.

Answer

The four terms form a strict containment hierarchy, where each inner set is a proper subset of the outer:

**AI but not ML** : MYCIN (1976), the medical diagnosis expert system that uses approximately 600 hand-coded IF-THEN rules. No parameter is learned from data; the rules are written by human experts. The system's performance does not improve with experience. This is purely symbolic AI.

**ML but not DL** : Support vector machines trained on hand-crafted image features (e.g., HOG descriptors), as used in pedestrian detection before 2012. The model _learns_ — it adjusts a linear boundary to classify examples correctly — but it uses a shallow, convex hypothesis space (a single linear separator in the kernel-induced feature space), not a deep neural network with multiple learned layers.

**DL but not LLM** : A ResNet-152 trained on ImageNet for image classification. This is a deep neural network with 152 layers, trained via gradient descent on labeled image data. It is not a language model — it processes images, not text, and does not generate sequences of tokens. It also does not use the Transformer architecture.

The key distinctions: AI is the broadest category (any system exhibiting intelligent behavior); ML adds the constraint of learning from data; DL adds the constraint of using deep neural networks; LLM adds the constraints of (1) operating on language and (2) using the Transformer architecture at scale.

**Q2.** State the three-element framework for machine learning. **Q3.** The knowledge acquisition bottleneck is the structural failure of symbolic AI. State it precisely: what does it mean, why does it occur, and why can't it be fixed by writing more rules?

Answer

The knowledge acquisition bottleneck is the difficulty of eliciting human expert knowledge into explicit, formal rules at the scale and precision required for a rule-based AI system to perform competently.

**What it means precisely:** A rule-based system needs rules. Rules must be: (1) explicit — a human expert must be able to state them; (2) formal — they must be unambiguous enough to be encoded as logical statements; (3) complete — they must cover the space of inputs the system will encounter; (4) consistent — no two rules should give contradictory outputs for the same input. In practice, achieving all four simultaneously for any non-trivial domain is impossible.

**Why it occurs:** Much human expertise is _tacit knowledge_ (Michael Polanyi's term) — "we know more than we can tell." A radiologist who can reliably identify subtle abnormalities in X-rays cannot necessarily describe, in words precise enough to encode as rules, what features they are attending to. The knowledge exists in the radiologist's visual system but not in a form that can be directly transcribed.

**Why more rules don't fix it:** Adding rules to handle cases the current system misclassifies creates three compounding problems:

  1. _Rule interaction_ : Each new rule potentially interacts with all existing rules. A system with nnn rules has O(n2)O(n^2)O(n2) potential rule interactions to check for consistency. At scale, this is unmanageable.
  2. _Brittleness at the boundary_ : A rule that correctly handles cases near its decision boundary will inevitably fail on cases that are slightly outside its design envelope — cases that differ from training examples in ways the rule-writer did not anticipate.
  3. _No self-correction_ : When a rule-based system makes an error, there is no mechanism for it to learn from that error and adjust its behavior. The correction must be made by a human, who must diagnose which rule was wrong and how to modify it without breaking other rules.



The fundamental contrast: machine learning systems improve automatically when exposed to more examples of the errors they make. Rule-based systems do not.

**Q4.** Connectionism went through three cycles of rise and fall before achieving its current dominance. For each cycle, state the key technical innovation that enabled the rise and the specific limitation that caused the fall.

Answer

**First cycle (rise: 1943–1958; fall: ~1969):**

  * _Rise_ : McCulloch-Pitts neuron (1943) — a mathematical model of a neuron as a binary threshold unit. Rosenblatt's perceptron (1958) — the first algorithm that could learn to classify linearly separable patterns from labeled examples.
  * _Fall_ : Minsky and Papert's mathematical analysis (1969) proved that single-layer perceptrons cannot represent the XOR function — a function that requires a non-linear decision boundary. Since every continuous function requires multiple layers to approximate, this was understood as showing that "neural networks" were fundamentally limited. (The analysis was correct for single-layer networks but did not address multi-layer ones.) Funding dried up.



**Second cycle (rise: 1986–early 1990s; fall: ~1995–2005):**

  * _Rise_ : Rumelhart, Hinton, and Williams (1986) — backpropagation demonstrated the possibility of training multi-layer networks, sidestepping the Minsky-Papert limitation. The universal approximation theorem — a two-layer network can represent any continuous function to arbitrary precision.
  * _Fall_ : Vanishing gradients prevented training networks deeper than two or three layers effectively. With only two or three useful layers, the "deep" in deep learning was not yet achievable. Simultaneously, support vector machines emerged with stronger theoretical guarantees and better empirical performance on datasets of the size available at the time.



**Third cycle (rise: 2006–present; no fall yet):**

  * _Rise_ : Hinton et al. (2006) — layer-wise pretraining initialized deep networks in a tractable region of parameter space. AlexNet (2012) — GPU-accelerated training, ReLU activations, dropout regularization, and massive labeled data (ImageNet) together solved the vanishing gradient and overfitting problems that had limited depth.
  * _Current limitation_ : High computational cost (training large models requires enormous energy and capital investment), opacity (the reasoning of deep networks is difficult to interpret), data hunger (deep networks require more labeled data than shallow methods), and brittleness to distribution shift. These are active research areas, not solved problems.

### Application Problems

**Q5.** You are asked to build an AI system for each of the following three tasks. For each task, recommend whether you would use symbolic AI, statistical machine learning (without deep learning), or deep learning, and justify your recommendation with reference to the three-element framework.

(a) A medical record system that automatically flags prescriptions for drug interactions, and must be able to explain every flag to a physician in terms of specific clinical guidelines.

(b) A system that reads satellite images of agricultural fields (1,000 labeled training images per crop type, 50 crop types) and estimates crop yield.

(c) A system that transcribes the spoken words in audio recordings of courtroom proceedings, where audio quality is variable and speakers have diverse accents.

Answer

**(a) Drug interaction flagging with explainability requirements:** **Recommendation** : Symbolic AI (rule-based system), supplemented by a curated knowledge base.

_Justification_ : The hypothesis space is effectively specified by clinical pharmacology guidelines — the set of known drug interactions is a finite, well-documented knowledge base. The loss function is implicitly the 0-1 classification loss on known interactions, but no learning from error is needed: the "ground truth" is the clinical literature. The optimization is trivial — there is no model to train. The critical constraint — every flag must be explainable in terms of specific clinical guidelines — is incompatible with learned models, whose decision boundaries cannot be directly interpreted as clinical rules.

The failure mode of statistical learning here is significant: a learned model might correctly flag an interaction but be unable to explain _why_ , which is unacceptable in a clinical setting where physician override and legal liability are at stake. The knowledge acquisition bottleneck is less severe here than in general medical diagnosis because drug interactions are systematically catalogued in pharmacological databases.

**(b) Crop yield estimation from satellite images:** **Recommendation** : Deep learning (convolutional neural network), likely with data augmentation given the limited training set.

_Justification_ : The input is unstructured (satellite imagery at various wavelengths), and the relevant features — vegetation density patterns, spectral signatures, spatial arrangement — are not easily expressible as hand-crafted rules or features. The hypothesis space needs to be large enough to capture the complex mapping from pixel values to yield estimates. 1,000 labeled images per crop type (50,000 total) is at the lower end for deep learning and may require transfer learning: initializing with a network pretrained on a large image dataset (e.g., ImageNet), then fine-tuning on the agricultural data.

Statistical machine learning with hand-crafted features (NDVI, texture statistics, spectral ratios) could also work and might outperform deep learning if the labeled data is genuinely scarce. A pragmatic approach: benchmark both and compare.

**(c) Courtroom speech transcription:** **Recommendation** : Deep learning (sequence-to-sequence model with attention, or a Transformer-based speech model).

_Justification_ : Speech transcription requires modeling two types of complex structure simultaneously: the mapping from acoustic features (spectrograms) to phonemes (learned perceptual representations), and the mapping from phoneme sequences to word sequences (learned language model). Neither is easily expressible as rules, and both benefit from training on very large datasets of transcribed speech. The diversity of accents and variable audio quality makes hand-crafted feature engineering impractical — the system must learn robust acoustic representations from data. Pre-trained speech models (Wav2Vec, Whisper) demonstrate that a single Transformer-based architecture trained on hundreds of thousands of hours of labeled audio achieves near-human accuracy on this task.

**Q6.** A researcher trains a linear regression model on a dataset of 100 examples and achieves a training mean squared error (MSE) of 2.3. When evaluated on a separate test set of 500 examples, the test MSE is 2.4. A colleague argues that the model is "clearly underfitting" because the training MSE is high. Another colleague argues the model is "clearly not overfitting" because the training and test MSE are nearly identical.

Evaluate both arguments using the three-element framework. Which, if either, is correct? What additional information would you need to determine whether the model is well-specified?

Answer

Both arguments contain a kernel of truth but are incomplete.

**The first argument ("clearly underfitting"):** The colleague is using "high training MSE" as evidence of underfitting. This is only meaningful in context. Underfitting — also called high bias — occurs when the hypothesis space is too small to capture the true relationship between inputs and outputs. An MSE of 2.3 is only "high" relative to some baseline. If the irreducible noise in the target variable has variance 2.0 (i.e., the best possible predictor could achieve MSE ≈ 2.0), then MSE 2.3 is only marginally worse than optimal and the model is well-specified. If the irreducible noise variance is 0.1, then MSE 2.3 represents a substantial fitting failure. The colleague is right to flag the possibility of underfitting but cannot confirm it without knowing the noise floor.

**The second argument ("not overfitting"):** This is correct and precisely stated, in the sense that _overfitting_ is defined by a gap between training and test performance. A gap of 0.1 between training MSE (2.3) and test MSE (2.4) is negligibly small, especially given the difference in dataset sizes (100 training, 500 test). However, "not overfitting" does not imply "well-specified." A model can achieve nearly identical training and test MSE while still performing poorly compared to the optimal predictor — this is precisely underfitting.

**Using the three-element framework:** The relevant decomposition is:

  * _Hypothesis space_ : Is the linear hypothesis space rich enough to capture the true relationship? If the relationship is genuinely nonlinear, no amount of data will reduce the training MSE below the approximation error of the linear class.
  * _Loss function_ : MSE is appropriate when the target variable is continuous and prediction errors are symmetric. In this case, it is the right metric.
  * _Optimization_ : With a linear hypothesis space and MSE loss, there is a closed-form solution (normal equations), so underfitting is not caused by optimization failure.



**Additional information needed** : (1) the MSE of a trivially constant predictor (predicting the training mean) — this establishes the baseline; (2) a plot of residuals vs. fitted values, which would reveal non-linearity in the residuals; (3) MSE of a richer model (e.g., a polynomial regression or a small neural network) on the same dataset — if a richer model achieves substantially lower training MSE without higher test MSE, then the linear model is indeed underfitting.

### Think Deeper

**Q7.** The three-element framework (hypothesis space, loss function, optimization algorithm) is proposed as a universal description of machine learning algorithms. But not all learning algorithms fit cleanly into this framework — consider k-nearest neighbors, which has no explicit loss minimization phase. More broadly, are there limits to the framework's universality? Can you think of a class of learning systems for which the three-element framework is a poor description?

Answer

The three-element framework is a very useful organizing principle, but its universality has genuine limits.

**k-Nearest Neighbors and instance-based learners:** As noted in Q2, k-NN stores the training data and classifies by local vote. There is no explicit loss minimization — the "hypothesis" is the entire training set, not a compact parameterization. One could argue that k-NN implicitly defines a hypothesis space (all Voronoi-partition classifiers induced by datasets of size nnn) and that "training" selects the hypothesis corresponding to the given data, but this is a stretch: the representation grows with the training set, and nothing is optimized. The framework applies only in a degenerate sense.

**Online/continual learning:** Systems that learn continuously as new data arrives (rather than on a fixed training set) fit awkwardly into the framework. The hypothesis at time ttt depends on the history of all data seen up to time ttt, and there is no single loss function being minimized over a fixed dataset. The optimization is typically greedy (each new observation triggers a local update), which may not minimize any global objective.

**Bayesian inference:** A Bayesian agent maintains a posterior distribution over hypotheses, P(θ∣data)∝P(data∣θ)P(θ)P(\theta | \text{data}) \propto P(\text{data} | \theta) P(\theta)P(θ∣data)∝P(data∣θ)P(θ). This is not an optimization problem — there is no single hypothesis selected. One can construct a point estimate (the MAP estimate) that fits the framework, but the full Bayesian update is richer: it maintains uncertainty over hypotheses rather than collapsing to a single one. The "hypothesis space" is the support of the prior; the "loss function" is the negative log-likelihood; but the "optimization" is replaced by integration (or its approximation).

**Evolutionary algorithms:** Evolution-inspired methods maintain a population of candidate solutions and select, recombine, and mutate them across generations. There is a fitness function (analogous to a loss), but the optimization is a stochastic search process without gradients. Many such algorithms do not fit the gradient descent paradigm at all.

**The deeper limitation:** The three-element framework implicitly assumes that learning reduces to optimization — that there is a fixed objective to be minimized. But many interesting learning systems involve exploration-exploitation tradeoffs (reinforcement learning), adversarial objectives (game theory, GANs), or no objective at all (unsupervised structure discovery). The framework is powerful within its domain — supervised gradient-based learning — but should be understood as a description of that domain rather than a universal theory of intelligence.

**Q8.** The history of AI shows three paradigms — symbolic, connectionist, statistical — competing and cycling, with each new wave not entirely replacing its predecessor. Statistical learning still outperforms deep learning on tabular data; symbolic methods still dominate where interpretability and logical correctness are required. Does this suggest that the three paradigms are not in competition but are instead each adapted to a different regime of problems? If so, what properties of a problem determine which paradigm is most appropriate?

Answer

The cycling history of AI paradigms does suggest that the three approaches are not simply successors but different tools optimally suited to different regimes. The evidence:

  * Symbolic methods still dominate in formal domains: theorem proving, formal verification, constraint satisfaction, planning in deterministic environments. These are exactly the domains where the "every aspect of learning can be precisely described" conjecture is true.
  * Statistical learning (random forests, gradient boosting) still dominates on tabular data with hundreds to thousands of labeled examples, where the inductive bias of tree-based partitioning matches the typical structure of such data better than the smooth parameterization of neural networks.
  * Deep learning dominates on high-dimensional unstructured data (images, audio, text) with millions or billions of training examples, where hand-crafted features are inadequate and the data volume justifies the computational cost.



**What properties determine the best paradigm?**

_Hypothesis space alignment:_ Symbolic methods work best when the structure of the correct function can be described formally (logical rules, grammars, constraint systems). Statistical methods work best when the function is "simple" by some measure (smooth, low-rank, tree-decomposable) and the training set is small enough that the statistical guarantees of these methods are binding. Deep learning works best when the function is complex and compositional, and enough data and compute are available to search the large hypothesis space effectively.

_Interpretability requirement:_ When the decision process must be auditable by humans (clinical decisions, legal decisions, financial decisions with regulatory oversight), symbolic methods or interpretable statistical models are preferred regardless of their raw performance. The "best predictor" is not the best system if its decisions cannot be inspected and challenged.

_Data availability:_ Symbolic methods require no labeled data (the rules come from experts). Statistical methods need hundreds to thousands of examples. Deep learning typically needs millions. This alone explains much of the empirical distribution of method choices across domains.

_Distribution shift sensitivity:_ Symbolic systems that encode domain knowledge explicitly can generalize to distribution shifts (new cases) if the shift preserves the logical structure of the domain. Statistical and deep learning models generally fail when the test distribution differs significantly from the training distribution — they have captured a correlation in the training data, not a causal mechanism. For applications where distribution shift is expected (e.g., deploying a model trained on 2020 data in 2025), the brittleness of purely data-driven methods is a serious concern.

The practical implication: the three paradigms should be seen as complementary tools in a methodology toolkit. The choice among them is a problem-specific engineering decision, not a philosophical commitment to a single view of intelligence.
