## Chapter Summary

  1. **The nested hierarchy** AI ⊃ ML ⊃ DL ⊃ LLM reflects an inheritance structure: each layer uses the mathematical tools of the layer below. This series builds the hierarchy from the bottom up.

  2. **Three paradigms** : Symbolic AI encodes knowledge as explicit rules — powerful where the domain is formally structured, fragile everywhere else. Connectionism encodes knowledge as distributed weights — learned from data, adaptable, but theoretically harder to analyze. Statistical learning provides theoretical guarantees about generalization — important for understanding _why_ a method works, not just _that_ it works.

  3. **The paradigm shift** : The move from hand-crafted rules and features to learned representations was enabled by three coincident factors: large labeled datasets, GPU-accelerated computation, and algorithmic innovations (ReLU, dropout, better initialization). Each of these was necessary; none was sufficient alone.

  4. **The three-element framework** : Every machine learning algorithm specifies a hypothesis space (what functions to consider), a loss function (what "wrong" means), and an optimization algorithm (how to search). This framework organizes the entire series. Every architecture is a hypothesis space; every training objective is a loss function; gradient descent is the universal optimizer.

  5. **The central question** : Why does the Transformer look the way it does? Answering this requires understanding every hypothesis class that came before — MLPs, CNNs, RNNs, LSTMs, word embeddings, encoder-decoder models, and the attention mechanism — because each Transformer component is the solution to a limitation of its predecessor. That story runs through the entire volume.
